About this page

MVT model

This page was made with Django, using MVT Model:



  • Model: Stores the information of the tables and instances.
  • View: Obtains the data from the Model and passes it to the Template. It also obtains user input from the Template, which it uses to update the data with the Model.
  • Template: What the user sees, displays data and reads user input.

Modelling

This is the ER diagram:



  • Degree: A degree can either be a bachelor or master’s degree (field with two options), and it also has a title and multiple subjects.
  • Subject: Each subject has a unique code and belongs to only one degree (although some subjects are in multiple degrees, they have different codes depending on the degree they’re associated with, so they’re technically different instances). Also, each subject is taught by multiple professors and each professor teaches multiple subjects.
  • Professor: Each of them has a link to their page in Lusófona and teaches one or multiple subjects.
  • Competence: The different skills a person has. Each technology requires one or more.
  • Technology: Includes name, logo, link to the official page and my personal level of interest/mastery of them, as well as the competences required for using them. Each technology requires one or more competences, and each competence can be required in one or more technologies.
  • Project: Each project is developed in one subject (foreign key). And requires specific competences and technologies. Initially, I thought I’d only include technologies as a many to many field, as they already have a many to many relationship with competences so it’d be redundant to also include competences; but I added them anyway because additional competences can be required for a project even if the technologies do not require them.
  • TFC: Some of the fields I included from the JSON are title, author, year, description, as well as degree as foreign key.
  • Education: Includes course name, and start/end year in order to be able to order it chronologically. I thought of including a foreign key to the degree, but I left it like that because there can be also external courses that are not in this database.
  • Making of: For each of the “steps” of the making of, I included a title, description and picture.
  • Language: An entity introduced by me because I thought it’s relevant to include the languages one speaks in the portfolio. It includes the name of the language and level of mastery (from 1, which is A1, to 7, which is native).
